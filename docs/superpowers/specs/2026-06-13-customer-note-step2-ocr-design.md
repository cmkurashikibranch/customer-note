前提：ステップ1（写真＋手入力版）全6タスク完了済み（`customer-note.html`）

## 目的と位置づけ

スマホ本体のOCR（iPhoneのテキスト認識／Googleレンズ）でコピーした名刺の文字を
アプリに貼り付け、各項目に自動で振り分けて入力の手間を減らす。
APIもOCRライブラリも使わず、完全オフライン・無料で動く「時短版」。

| ステップ | 入力方式 | API | 状態 |
|---------|---------|-----|------|
| 1 | 写真＋手入力 | 不要 | 完了 |
| 2 | スマホOCR貼り付け＋自動振り分け | 不要 | **本設計書の対象** |
| 3 | AI自動読取（Claude Haiku、GAS経由） | 約0.3〜0.5円/枚 | ゴール |

データの流れ（OCRはスマホ任せ＝アプリ側はテキスト解析のみ）：

```
 ┌─ スマホ本体でOCR ────┐      ┌──── お客様ノート ────┐
 │ ①名刺を写す          │      │ ④貼り付け欄にペースト │
 │ ②テキスト認識/Lens   │ ──▶ │ ⑤[読み取る]で振り分け │
 │ ③コピー              │ copy │ ⑥確認・修正して保存   │
 └────────────────┘      └──────────────────┘
```

## 解析ロジック `parseCard(text)`

貼り付けた文字を1行ずつに分解し、**確実な順**に各項目へ振り分ける純関数。
DOMに触れない（文字列 → `{ name, company, phone, email, memo }` を返す）。

### 前処理
- 全角英数字・記号（＠ や全角数字）を半角へ正規化してから解析
- 改行で分割 → 各行を trim → 空行を除去

### 振り分けルール

| 項目 | 取り方 | 確度 |
|------|--------|------|
| メール | 正規表現 `[\w.%+-]+@[\w.-]+\.[A-Za-z]{2,}` の最初の一致 | ◎ ほぼ確実 |
| 電話 | 0始まり・数字10〜11桁。携帯(070/080/090)やTEL表記を優先、FAX行は除外 | ○ 高い |
| 会社 | 「株式会社/(株)/有限会社/合同会社/法人/工務店/事務所/商店/製作所/医院/薬局/店/屋/Co./Inc./Ltd.」等を含む行 | ○ 高い |
| 役職 | 「代表/取締役/社長/専務/常務/部長/課長/係長/主任/室長/店長/施設長/管理者/理事/会長/マネージャー/CEO」等を含む行 → **メモへ** | △ 補助 |
| 名前 | 上記に当たらない行から推測（かな/漢字中心・2〜6字・姓名スペース行を優先・数字を含まない） | △ 外すことあり |
| 残り | 住所・〒・FAX番号・URL など、どれにも当たらない行 → **メモへ** | 取りこぼさない |

- **名前は「外すことあり」と割り切る。** だからこその確認画面。当たらなければ空欄で人が入れる
- メール・電話は最初の1件を採用。2件目以降や FAX 番号は memo へ送り、取りこぼさない

### メモの組み立て
1. 役職を検出したら先頭に `役職：◯◯`
2. 続けて、どの項目にも振り分けられなかった残り行を改行でつなぐ（住所・〒・FAX・URL等）

```
役職：代表
〒700-0000 倉敷市○○町1-2-3
FAX 086-xxx-xxxx
```

保存前にユーザーがここへ「守成クラブで交換」などの出会いメモを書き足す想定。

## 確認・保存の流れ

```
貼り付け → [読み取る] → 名前/会社/電話/メール/メモに反映 → 目で確認・手で修正 → [保存する]
```

- `[読み取る]` は **名前・会社・電話・メール・メモの5項目を貼り付け内容から作り直す**
  （押すたび再解析。既存の手入力があれば上書きされる。確認はそのあと）
- 貼り付け欄が空のまま押したら「貼り付け欄が空です」とだけ案内して何もしない
- 保存処理（`saveCustomer`）・容量警告・履歴生成はステップ1のまま流用（変更なし）

## テスト方針

- `parseCard` を **純関数**として実装し、実際の名刺を想定したサンプル文字列で
  **Nodeの単体テスト**にかける（`node --test tests/parse-card.test.js`）
- 単一HTMLは維持する。HTML内の解析ブロックを `/* PARSER:START */ … /* PARSER:END */`
  マーカーで囲み、テスト側がその区間を読み取って評価する（**1つのソースを共有しドリフトを防ぐ**）
- UI（貼り付け→読み取り→反映→保存）は最後にブラウザで確認
  （単一HTMLはローカルHTTP経由で開く。ステップ1の学びに準拠）
- テストするサンプル：標準的な縦並び名刺／TELとFAX両方ある名刺／メールなし名刺／
  役職つき名刺／OCRが全角になったケース／ほぼ空文字
