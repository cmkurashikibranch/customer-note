# お客様ノート 設計書（ステップ2：OCR貼り付け＋自動振り分け）

作成日：2026-06-13
ステータス：**v2 — UI/UX・エンジニア2レビュー反映済み**（承認待ち）
前提：ステップ1（写真＋手入力版）全6タスク完了済み（`customer-note.html`）

## 目的と位置づけ

スマホ本体のOCR（iPhoneのテキスト認識／Googleレンズ）でコピーした名刺の文字を
アプリに貼り付け、各項目に自動で振り分けて入力の手間を減らす。
APIもOCRライブラリも使わず、完全オフライン・無料で動く「時短版」。

| ステップ | 入力方式 | API | 状態 |
|---------|---------|-----|------|
| 1 | 写真＋手入力 | 不要 | 完了 |
| 2 | スマホOCR貼り付け＋自動振り分け | 不要 | **本設計書の対象** |
| 3 | AI自動読取（Claude Haiku、GAS経由） | 約0.3〜0.5円/枚 | ゴール |

データの流れ（OCRはスマホ任せ＝アプリ側はテキスト解析のみ）：

```
 ┌─ スマホ本体でOCR ────┐      ┌──── お客様ノート ────┐
 │ ①名刺を写す          │      │ ④貼り付け欄にペースト │
 │ ②テキスト認識/Lens   │ ──▶ │ ⑤[読み取る]で振り分け │
 │ ③コピー              │ copy │ ⑥確認・修正して保存   │
 └────────────────┘      └──────────────────┘
```

| やること | やらないこと |
|---------|-------------|
| 貼り付け欄＋[読み取る]ボタンを登録画面に追加 | アプリ内OCR（Tesseract等） |

| 解析を純関数に切り出しNodeでテスト | 完全自動保存（必ず人が確認） |
| | データ構造の変更 |

- **いじるのは登録画面まわり＋詳細画面の履歴表示の軽微修正のみ。** 保存ロジック・データ構造は据え置き。
- **非日本語の名刺は best-effort**（メール・電話・Inc.等の会社は拾える範囲で。名前は拾えれば拾う）。
  完全対応はステップ3のAIに委ねる。**「例外を投げず最後まで動く」ことだけを保証**する。

## 解析ロジック `parseCard(text)`

DOMに触れない**純関数**（文字列 → `{ name, company, phone, email, memo }`）。
副作用ゼロ・冪等（同じ入力なら同じ出力）。上書き判断はこの関数の外（DOM反映層）に置く。

### 前処理：全角→半角の正規化（電話判定の生命線）

| 種別 | 対象 | 半角化先 |
|------|------|---------|
| 英数 | Ａ-Ｚ ａ-ｚ ０-９ | A-Z a-z 0-9 |
| アット | ＠(U+FF20) | @ |
| ドット | ．(U+FF0E) | . |
| ハイフン類 | ー(U+30FC) －(U+FF0D) ‐(U+2010) ―(U+2015) −(U+2212) | - |
| 電話記号 | ℡(U+2121) | TEL |
| 括弧 | （ ） | ( ) |
| 空白 | 全角SP(U+3000) | 半角space |

- **中黒「・」はハイフンに含めない**（氏名区切りで使われるため）
- 冒頭で `text.length > 5000` なら先頭5000字に切る（OCR暴発入力のフリーズ防止）

### 確定的パイプライン（順序が仕様。各行に「消費済み」フラグ）

```
1. 正規化（上表）
2. 改行分割 → 各行 trim → 空行除去
3. email抽出  : 全文から正規表現。最初の1件→email。末尾の非英字をstrip。該当行を消費
4. 除外行マーク: 〒/郵便番号・住所語・URL を含む行 → memo候補（電話/会社/名前の対象外）
5. phone/fax  : 未消費・未除外行からトークン単位で抽出。tel優先1件→phone。他→memo候補
6. 会社        : 未消費行からキーワード一致の最初の1行→company（役職判定より先）
7. 役職        : 未消費行からキーワード一致→「役職：◯◯」をmemo先頭へ。
                同一行に名前混在なら役職語を除いた残りを名前候補に回す
8. 名前        : 未消費行をスコアリングし最高得点→name（無ければ空）
9. memo組み立て: 役職 → 未消費の残り行（住所/〒/FAX/URL等）を改行連結
```

### 各ステップ詳細

**③ email** … `/[\w.%+-]+@[\w-]+(?:\.[\w-]+)+/`（複数ドット対応で `co.jp` を取りこぼさない）。
マッチ後 `.replace(/[^A-Za-z]+$/,'')` で末尾の全角句点・括弧を除去。

**④ 除外行** … `〒` or `郵便番号`、または住所語 `都|道|府|県|市|区|町|村|丁目|番地|号`、
または `https?://` `www.` を含む行。最優先で memo 候補にし、以降の電話・会社・名前の対象から外す
（番地の数字が電話に化けるのを構造的に防ぐ）。

**⑤ phone/fax**（行ではなくトークン単位）…
1. 候補抽出 `/(?:\+81|0)[\d\-()\s]{8,13}\d/g`
2. 各候補を digits 化：`replace(/[^\d]/g,'')` → 先頭 `+81`/`81` は `0` に正規化
3. 採用条件 `/^0\d{9,10}$/`（10〜11桁。`0120` も10桁で合致）
4. 直前ラベルで分類：`FAX|ファ?ックス`→fax ／ `TEL|℡|電話|携帯|Mobile|Tel`→tel(優先) ／ ラベル無し→tel(低優先)
5. tel候補の最初の1件→`phone`（**ハイフン整形した数字列**で返す＝`tel:`安全）。
   残りの番号・fax→memo候補。**telが無くfaxのみなら phone は空、fax は memo へ**

**⑥ 会社** … キーワード初期リスト（下記）一致の最初の未消費行→company。

**⑦ 役職** … キーワード初期リスト一致行を検出。
`代表取締役 山田太郎` のように**役職語＋人名が同一行**なら、役職語にマッチした部分を取り除いた
残り（2〜6字の連続）を名前候補へ。役職は `役職：◯◯` として memo 先頭に置く。

**⑧ 名前（スコアリング）** … 未消費行を採点し最高得点を name に：
`漢字を含む +2 ／ かなのみ -1 ／ 英字2語(姓名) +1 ／ 数字を含む =除外 ／ 会社・役職語を含む =除外 ／ 長さ2〜6 +1 ／ 姓名スペースあり +1`。
**最頻パターン（漢字フルネーム・姓名空白あり）は確実に当てる**ことをテストで担保。該当なしは空。

**⑨ memo** … `役職：◯◯`（あれば）→ 未消費の残り行を `\n` 連結。

## 反映・確認・保存の流れ

```
貼り付け → [読み取る] → 確認 → 名前/会社/電話/メール/メモへ反映 → 目で確認・手で修正 → [保存する]
```

- **[読み取る] 押下時**：
  - 貼り付け欄が空 → 「貼り付け欄が空です」と案内して何もしない
  - フォーム5項目のいずれかに入力あり → `confirm('入力済みの内容を読み取り結果で置き換えますか？')`。
    キャンセルなら何もしない
  - 全空欄 or 確認OK → 反映（下記）
- **反映ルール（事故防止）**：
  - 名前・会社・電話・メール：**parseCard が非空を返した項目だけ置換**（空を返した欄は既存の手入力を消さない）
  - メモ：**追記マージ**（既存メモの行を保持し、parseが返した行のうち未収録のものだけ追加）。
    ユーザーが書いた「守成クラブで交換」等は消えない
  - 推測で入った名前・会社の欄に「推測」マーカーを付ける（編集 or 保存で解除）
- **読み取り後の状態メッセージ**（aria-live・常駐）：
  - 基本：「✨ 読み取りました。電話とメールはそのまま、お名前と会社は念のため確認してくださいね」
  - 電話・メールが両方空：「（電話・メールは見つかりませんでした。手で入れてくださいね）」を併記
  - 名前が空：「（お名前は読み取れませんでした。上の欄にご記入を）」を併記
- **クリア**：貼り付け欄＋自動反映した5項目＋推測マーカーをリセット（**写真は残す**）
- 保存処理（`saveCustomer`）・容量警告・履歴生成・`regPhotoBusy`ガードはステップ1のまま流用（変更なし）

## テスト方針

- `parseCard` を**純関数**として実装し、実際の名刺を想定したサンプル文字列で **Nodeの単体テスト**にかける
- 単一HTMLを維持。HTML内の解析ブロックを `/* PARSER:START */ … /* PARSER:END */` マーカーで囲み、
  **そのブロックは外部識別子を一切参照しない自己完結ブロック**とする（ヘルパー・正規表現も内包）。
  ブロック末尾に次のガードを置く：
  ```js
  if (typeof module !== 'undefined' && module.exports) module.exports = { parseCard };
  ```
  ブラウザでは `typeof module === 'undefined'` で無害。**単一HTML・外部依存なし・オフラインを一切破らない**
- テスト側は `eval` を使わず、マーカー区間を読み取り **`new Function` で安全に取り出す**（グローバル非汚染）：
  ```js
  const block = html.match(/\/\* PARSER:START \*\/([\s\S]*?)\/\* PARSER:END \*\//)[1];
  const mod = { exports: {} };
  new Function('module', 'exports', block)(mod, mod.exports);
  const { parseCard } = mod.exports;
  ```
- 実行コマンド：`node --test tests/parse-card.test.js`（ディレクトリ指定はMODULE_NOT_FOUNDになる環境のため**ファイル直接指定**）
- UI（貼り付け→読み取り→確認→反映→保存）は最後にブラウザで確認（単一HTMLは**ローカルHTTP経由**で開く）

### 推奨テストケース（期待値まで固定する）

```
[正常系]
 1. 標準縦並び（漢字フルネーム＋(株)＋TEL＋mail）→ 全項目命中
 2. 役職＋名前同一行「代表取締役 山田太郎」→ name=山田太郎, memo先頭=役職：代表取締役
 3. TEL/FAX両方同一行 → phone=TEL側, memo に FAX 番号
 4. 携帯のみ 090… → phone採用
 5. 0120 / +81 → 正規化して採用
[誤検出回避]
 6. 〒700-0000＋番地1-2-3 の住所行 → phone に化けない・memoへ
 7. ℡記号(U+2121) → TELとして認識
 8. 全角数字・＠・全角ドットのメール/電話 → 正規化して命中
[エッジ]
 9. メールなし名刺 → email='' で他は命中
10. FAXのみ（telラベルなし）→ phone='' , memo にFAX
11. 空文字/空白のみ → 全項目'' ・例外を投げない
12. 一致ゼロ（記号の羅列）→ 全項目'' ・落ちない
13. 英語のみ名刺 → email/phone/company命中、name は best-effort
14. 超長文(5000字超) → 切り詰めて完走（フリーズしない）
15. 冪等性：同じ入力を2回 → deepEqual（純関数・副作用ゼロの担保）
```
